## 2024-05-18 - FlatList Nested in ScrollView Anti-Pattern
**Learning:** Nesting a `<FlatList>` with `scrollEnabled={false}` inside a `<ScrollView>` breaks the internal virtualization of the FlatList. React Native renders all elements of the FlatList immediately because the parent ScrollView forces it to expand to its full height.
**Action:** Always use a single root `<FlatList>` and utilize `ListHeaderComponent` and `ListFooterComponent` for any UI elements that should scroll along with the list.
## 2026-07-03 - Testing Inner Logic in Custom Test Harnesses \n**Learning:** When a repository uses custom test runners without module mocking (like simple `node test.js`), inner stateful functions from hooks can be tested by replicating them in the test file and passing dependencies (like state or external constants) as explicit function parameters. \n**Action:** When adding tests to custom plain-JS test files, replicate the target function, parameterize its external dependencies, and test it as a pure function instead of trying to mock the module system.
## 2026-07-03 - Avoiding JSX.Element returns \n**Learning:** Explicitly declaring a component's return type as `: JSX.Element` can cause `Cannot find namespace 'JSX'` errors in some configurations (especially React Native / Expo with strict TypeScript setups). \n**Action:** Let TypeScript infer the return type of React components automatically rather than typing them explicitly.
## 2026-07-03 - Array Index Tuple Types \n**Learning:** When using `as string` on array accesses like `segments[1] as string | undefined`, TypeScript might infer the array as a tuple of length 1 (e.g. `[string]`) and throw `Tuple type '[string]' of length '1' has no element at index '1'`. \n**Action:** Use a type assertion on the array itself, like `(segments as string[])[1]`, or rely on a length check before accessing an index to resolve these strict tuple type inferences.
