## The 10, expanded

### 01 · Quest! — a real-life adventure (Build Log series opener)
**Pillar 01 · Reel · Week 1 anchor**

This is episode one of your recurring build log, not a one-off. Structure it as a cold open on the app, then you.

**The beat sheet (30–45s reel):**
1. Hook (0–3s): screen recording of Quest! actually working — the most "wait, that's real?" moment you have (badge unlock animation? map view?)
2. "I'm a CS student and I've spent [n] months building a real-life adventure app."
3. Three fast cuts: what it does (10s max — quests, badges, moderation, whatever demos best)
4. One honest confession: the mistake that cost you the most time
5. CTA: "I'm documenting the whole build here. Episode 2: the feature that nearly broke me."

**Hooks to test:**
- "I turned real life into a video game. Here's the app."
- "Every CS student says they're building an app. I actually shipped mine."
- "Month [n] of building an app that makes real life playable."

**Design/style:** film raw, but the title card and end card use the brand — Archivo Black "QUEST! · BUILD LOG · EP.01" on cream, torn-paper edge, one rainbow stripe divider. The contrast of raw screen recording + designed cards IS the look.

**What you need to do:** pick the single most impressive 3-second screen recording you can capture from the current build. Make the EP.01 title card template once — you'll reuse it every week. Write down the honest mistake before filming; specificity ("I rebuilt the badge system three times because I didn't design the unlock thresholds first") beats vague ("I made mistakes").

### 02 · The 4-stage AI pipeline that codes, tests, and reviews itself
**Pillar 01 · Spec Sheet reel + companion carousel · Week 2 flagship**

Your Planner → Coder → Tester → Reviewer Claude Code setup, presented like a Porsche 962 spec poster. This is the post that makes engineers follow you.

**Reel beat sheet (45–60s):**
1. Hook (0–3s): "I don't write most of my code anymore. My pipeline does." over a sped-up terminal recording
2. The spec card appears — exploded-view diagram, four stages labeled in Space Mono
3. Walk each stage in one sentence each: Planner writes the spec → Coder implements it → Tester writes and runs tests → Reviewer signs off
4. Show a REAL run: feature request in, working code out (timelapse)
5. The honest caveat: "Here's the one thing it still gets wrong…" (credibility moment)
6. CTA: "Full breakdown in the carousel. Want the setup guide? Comment PIPELINE."

**Carousel (Spec Sheet format, 7–8 slides):**
- S1: Hero — pipeline diagram as motorsport spec card, "AGENT PIPELINE MK.I" in Archivo Black
- S2–5: One stage per slide, each with a mono stat block (what it reads, what it writes, what it's not allowed to touch)
- S6: STATS block — time saved per feature, what broke, lines of config
- S7: "Where it bites you" — the failure mode
- S8: CTA card

**Hooks to test:**
- "My code reviews itself before I ever see it."
- "4 AI agents. 1 feature request. 0 hands on keyboard."
- "This is what a one-person engineering team looks like in 2026."

**What you need to do:** screen-record one clean end-to-end pipeline run on a real Quest! feature (pick a small, visual one). Build the Spec Sheet carousel template — this template gets reused for ideas 03 and every future feature ship. The "comment PIPELINE" CTA needs a ManyChat-style auto-DM or just a pinned comment with a link — decide before posting.

## Production system (so this doesn't eat your life)

**Build 4 templates once, in week 0/1 — everything refills them:**
1. **Spec Sheet** (carousel + reel end-card) — ideas 02, 03, every future ship
2. **Field Notes** (carousel) — ideas 04, 05, all future teach posts
3. **THEN/NOW** (split reel frame + numbered end card) — idea 06 and the whole franchise
4. **Series assets** — Build Log EP.## title card, the three verdict stamps

**Batching rhythm:**
- One design afternoon per week: next week's carousels + title cards
- One filming session per week: both reels back-to-back (same setup, same light)
- Screen recordings: capture continuously while you work — a `content/raw` folder of clips means you're never recreating a demo. You're building anyway; keep OBS running.

**Caption formula (every post):** Hook line (repeat/extend the on-screen hook) → 2–4 sentences of the real substance (teach in the caption, not just the visual — IG reads captions for ranking now) → one question to the audience → CTA (save/share/comment keyword).

**Measurement that matters in month one:** ignore follower count. Watch: saves on carousels (is Teach working?), shares on THEN/NOW (is Signal spreading?), comments on the tier list (is the audience real?), and profile-visits-per-reel (is the hook attracting the right people?). Whichever format over-performs, double it in August.
